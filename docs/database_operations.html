Database Operations Documentation

Data Types

User

  • id?: number - Unique identifier
  • chinese_name: string - User's Chinese name
  • email: string - User's email address
  • phone?: string - User's phone number

VolunteerEvent

  • id?: number - Unique identifier
  • title: string - Event title
  • startDate: string - Event start date
  • endDate: string - Event end date
  • location: string - Event location
  • maxParticipants: number - Maximum number of participants
  • registrationDeadline: string - Registration deadline
  • projectManagerName: string - Project manager's name
  • projectManagerEmail: string - Project manager's email
  • projectManagerPhone: string - Project manager's phone
  • description?: string - Event description
  • image?: string - Event image URL
  • participants?: number - Current number of participants
  • projectManagerTitle?: string - Project manager's title
  • projectManagerLine?: string - Project manager's Line ID
  • category?: string - Event category
  • difficulty?: string - Event difficulty level
  • requirements?: string[] - Participation requirements
  • benefits?: string[] - Event benefits
  • items?: string[] - Required items
  • notes?: string[] - Additional notes
  • transportation?: string - Transportation information
  • meetingPoint?: string - Meeting point details
  • schedule?: string - Event schedule
  • status?: string - Event status
  • created_at?: string - Creation timestamp
  • updated_at?: string - Last update timestamp
  • createdBy?: number - Creator's user ID

Operations

User Operations

create(user: User)

Creates a new user in the database.

Event Operations

create(event: VolunteerEvent)

Creates a new volunteer event.

getAll()

Retrieves all volunteer events.

getById(id: number)

Retrieves a specific event by its ID.

update(id: number, event: Partial)

Updates an existing event. Returns boolean indicating success.

updateParticipants(id: number, count: number)

Updates the participant count for an event.

delete(id: number)

Soft deletes an event. Returns boolean indicating success.

User Event Operations

register(userId: number, eventId: number)

Registers a user for an event.

getUserEvents(userId: number)

Retrieves all events a user has registered for.

getEventUsers(eventId: number)

Retrieves all users registered for an event.

Implementation Notes

  • All database operations are implemented using SQLite through better-sqlite3
  • Array fields (requirements, benefits, items, notes) are stored as JSON strings in the database
  • Dates are stored in ISO 8601 format
  • Soft deletion is implemented for events (records are marked as deleted rather than removed)
  • Automatic timestamp management for created_at and updated_at fields
